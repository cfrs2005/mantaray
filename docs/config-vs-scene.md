# 配置模式 vs 场景模式

这是使用本项目前**必须理解**的概念。

## 两种模式

代理客户端通常提供两种流量处理方式：

### 配置模式（Configuration）

- 根据 **配置文件中的规则** 决定流量走向
- 规则写在 `.conf` 文件中，可版本化、可审计
- 行为完全由文件内容决定，**可预测、可解释**

### 场景模式（Scene）

- 根据 **客户端内预设的场景规则** 决定流量走向
- 规则存储在 app 内部，不易导出和版本管理
- 可能根据网络环境（Wi-Fi / 蜂窝）自动切换行为

## 本项目使用配置模式

原因：

1. **可版本化** — 所有规则在 GitHub 上可追溯
2. **可解释** — 每条规则写在文件里，命中逻辑透明
3. **可复现** — 同一份配置在任何设备上行为一致
4. **可审计** — 出问题时能定位到具体哪条规则

## 什么时候不要用场景模式

- 当你需要精确控制某个域名的出口时
- 当你需要确保高风控服务的出口一致性时
- 当你需要排查「某个请求为什么走了代理/直连」时
- 当你需要在多设备间同步规则时

## 如何确认当前模式

在客户端首页查看当前生效的模式标识。如果显示的是某个场景名称而非配置文件名称，说明你正在使用场景模式，需要切换。

## 常见误区

| 误区 | 事实 |
|------|------|
| 场景模式更智能 | 场景模式更难追踪，不等于更智能 |
| 配置模式需要经常改 | 好的配置可以长期不动 |
| 两种模式效果一样 | 规则来源不同，行为可能不同 |
| 场景模式更安全 | 不可解释的行为反而增加风险 |
