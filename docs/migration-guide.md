# 迁移指南

从其他配置迁移到 Mantaray。

## 迁移前

1. **备份当前配置** — 导出当前正在使用的配置文件
2. **记录当前节点** — 记下你常用的节点名称和地址
3. **列出自定义规则** — 如果你有手动添加的规则，记录下来

## 迁移步骤

### 1. 导入 Mantaray 配置

复制日常模式 URL 导入：

```
https://raw.githubusercontent.com/cfrs2005/mantaray/main/configs/daily.conf
```

### 2. 选择节点

Mantaray 配置中的 `PROXY` 策略需要你自行选择节点。导入后在客户端中为 PROXY 策略指定你的节点。

### 3. 迁移自定义规则

如果你之前有自定义规则：

- 国内直连域名 → 添加到 `rules/direct_cn.list`
- 海外代理域名 → 添加到 `rules/proxy_global.list`
- 高风控服务 → 添加到 `rules/sensitive_services.list`
- 其他直连规则 → 添加到 `rules/custom.list`

### 4. 验证

- 访问国内站点，确认直连
- 访问海外站点，确认走代理
- 访问敏感服务，确认出口 IP 符合预期

## 从复杂配置迁移

如果你之前的配置有大量策略组、自动选择、回退节点等：

1. **不要试图复刻** — Mantaray 的设计目标是简单稳定
2. **先用日常模式跑几天** — 观察是否有异常
3. **按需添加规则** — 发现缺失再补，不要一次性导入大量旧规则
4. **放弃自动选择** — 至少在高风控场景下，手动固定节点

## 回滚

如果 Mantaray 不适合你的需求，随时可以切回旧配置。建议保留旧配置文件备份。
